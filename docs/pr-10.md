# PR10: 实时对话中"动态画面"——问完问题自动截最新画面

## 背景

PR8 的实现里，Qwen Realtime 会话的"当前画面"是会话开始时上传的 1 张截图。
AI 在用户后续对话中看到的**始终是这张旧图**，无法回答"我手里拿的是什么"
"现在能看到吗"这种依赖**提问瞬间**画面的问题。

PR10 把"AI 看到的画面"从「会话开始时的旧图」改成「**用户每次提问时的最新截图**」，
同时不增加持续视频流、不打破 PR8 的实时语音链路、不破坏 PR9 的视觉缓存。

## 设计目标

1. **每次提问 → 自动截一张最新图**（仅在用户问视觉相关问题时）
2. **手动按钮兜底**：用户可以随时点 "📷 重新截取画面"
3. **不动 PR8 实时音频链路**、**不动 PR9 视觉缓存**
4. **不增加任何视频流上传**
5. **不写盘**：新图替换内存中旧图，旧图立即释放

## 架构变化

### 之前的链路
```
Qwen Realtime
   ├─ input_audio (用户语音)
   ├─ analyze_current_frame (Qwen 工具)
   └─ 工具调用 → 后端取 session._frame_bytes (会话开始时的旧图) → 豆包
```

### PR10 之后的链路
```
Qwen Realtime
   ├─ input_audio (用户语音)
   ├─ input_audio_transcription.completed (用户一句话转写完)
   │     └─ 前端 shouldCaptureFrame(transcript) ?
   │           ├─ 是 → 截新图 → session.update_image → 后端替换 frame_bytes
   │           └─ 否 → 跳过
   ├─ analyze_current_frame (Qwen 工具)
   └─ 工具调用 → 后端取 session._frame_bytes (现在是最新的图) → 豆包
```

**关键不变量**：
- 图片不直接发给 Qwen（保持"图片只是 Qwen 工具的输入"原则）
- 图片替换是进程内事件，**不写盘**
- 不上传连续视频流
- 单次会话只会多消耗：每次"视觉相关"提问多 1 张截图（已压缩到 ≤2 MB）

### 手动按钮

实时对话区新增 `📷 重新截取画面` 按钮。点击后：
- 立即截图 + 压缩
- 立即发 `session.update_image` 消息
- 提示"📸 画面已更新（手动）"

适用于：用户发现 Qwen 引用了旧图 / 用户换了物体 / 用户没说话但要更新画面。

## WebSocket 协议

### 客户端 → 服务端
```json
{
  "type": "session.update_image",
  "image_base64": "...",
  "content_type": "image/jpeg"
}
```

### 服务端 → 客户端（成功）
```json
{
  "type": "session.image_updated",
  "size_bytes": 12345
}
```

### 服务端 → 客户端（失败）
```json
{
  "type": "session.error",
  "message": "图片超过 2 MB"
}
```

校验规则：
- 必须是合法 base64
- 解码后不能为空
- 大小 ≤ 2 MB
- content_type 必须在 `image/jpeg` / `image/png` / `image/webp` 之中

## 后端实现

### `QwenRealtimeSession.update_frame()`

```python
async def update_frame(
    self,
    *,
    image_base64: str,
    content_type: str = "image/jpeg",
    max_size_bytes: int = 2 * 1024 * 1024,
    allowed_types: Optional[set] = None,
) -> int:
    """替换当前会话的内存截图（PR10）。
    
    - 不会发图片给 Qwen，只在下次工具调用 analyze_current_frame 时使用
    - 不写入磁盘
    - 不影响正在进行的音频上行
    """
    if self._closed:
        raise ValueError("会话已关闭")
    b64 = (image_base64 or "").strip()
    if not b64:
        raise ValueError("缺少 image_base64")
    try:
        data = base64.b64decode(b64, validate=True)
    except Exception as e:
        raise ValueError(f"image_base64 无法解析: {type(e).__name__}") from e
    if not data:
        raise ValueError("图片内容为空")
    if len(data) > max_size_bytes:
        raise ValueError(f"图片超过 {max_size_bytes // (1024 * 1024)} MB")
    ct = (content_type or "image/jpeg").lower().strip()
    if allowed_types and ct not in allowed_types:
        ct = "image/jpeg"
    # 替换为新图片（旧的立即释放）
    self._frame_bytes = b""
    self._frame_bytes = data
    self._content_type = ct
    return len(data)
```

### `main.py` 路由

在 WebSocket 消息循环中新增 `session.update_image` 分支：

```python
if ptype == "session.update_image":
    if not initialized or qwen_session is None:
        await forward_event({"type": "session.error", "message": "会话尚未初始化，无法更新画面"})
        continue
    b64 = (payload.get("image_base64") or "").strip()
    ct = (payload.get("content_type") or "").lower().strip()
    if not ct or ct not in REALTIME_ALLOWED_IMAGE_TYPES:
        ct = REALTIME_DEFAULT_CONTENT_TYPE
    try:
        new_size = await qwen_session.update_frame(
            image_base64=b64,
            content_type=ct,
            max_size_bytes=REALTIME_MAX_IMAGE_SIZE_BYTES,
            allowed_types=REALTIME_ALLOWED_IMAGE_TYPES,
        )
        await forward_event({"type": "session.image_updated", "size_bytes": new_size})
    except ValueError as e:
        await forward_event({"type": "session.error", "message": str(e)})
    except Exception as e:
        logger.warning("update_frame failed: %s", e)
        await forward_event({"type": "session.error", "message": f"更新画面失败：{type(e).__name__}"})
    continue
```

## 缓存 & 去重的协作

PR10 **不改变** PR9 的缓存 Key（SHA-256(图片) + 标准化问题）。

举例：
1. 用户问"我手里拿的是啥？" → 自动截 → 调豆包 → 结果缓存
2. 用户再问一次相同问题（画面没变） → 命中缓存，0 毫秒
3. 用户换了手里的东西 → 自动截新图 → SHA-256 改变 → 新 Key → 调豆包
4. 用户说"今天天气怎么样" → 关键词不命中 → 不截 → 调 Qwen 但 Qwen 不会调工具
5. 用户问"画面里能看到啥" → 截 → 调豆包

## 局限性 / 权衡

| 限制 | 原因 | 影响 |
|---|---|---|
| 关键词判定不完美 | 不引入 LLM 做意图识别（成本） | "我刚买了啥"这种视觉词可能漏 |
| 用户"嗯"、"啊"停顿时也会触发（如果含视觉词） | 关键词匹配而非语义理解 | 偶尔有"过度截图"，但 1 张 ≈ 几百 KB 内存 |
| 5 分钟内同一图同一问题可命中缓存 | PR9 缓存设计 | 重复提问体验好；过期后正常重调 |
| 服务重启后画面丢失 | 不落盘 | 重新进会话时仍可正常 `session.init` |
| 不支持图片相似度合并 | 不引入算法 | 不同 SHA-256 视为不同图 |
