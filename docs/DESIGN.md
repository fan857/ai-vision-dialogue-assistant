# 设计文档

## 一、用户故事

### 计划实现

1. 用户可以打开摄像头，让 AI 看见当前画面。
2. 用户可以打开麦克风，通过语音向 AI 提问。
3. AI 能结合画面内容和用户问题进行回答。
4. AI 回复可以显示在页面上。
5. AI 回复可以通过语音朗读出来。
6. 系统能控制视觉模型调用成本。

## 二、成本控制策略

### 计划采用

1. 不上传连续视频流，只在用户提问时截取当前帧。
2. 图片上传前进行压缩。
3. 只在问题涉及画面时调用视觉模型。
4. 多轮对话中复用最近一次视觉摘要。
5. 对请求进行节流，避免频繁调用模型。
6. API Key 放在环境变量中，不提交到仓库。

## 三、系统架构

前端：
- 摄像头采集
- 麦克风采集
- 语音识别
- 截图上传
- AI 回复展示
- 语音朗读

后端：
- 接收用户文本和图片
- 调用多模态模型
- 返回 AI 回复
- 管理环境变量和成本控制逻辑
