# 七牛云 XEngineer 最后一批实训项目规则

## 项目背景

本项目参加七牛云 × XEngineer 暑期实训营最后一批议题。

所选题目：题目一：AI 视觉对话助手。

题目要求：
开发一款与 AI 对话的应用。用户可以打开摄像头与麦克风，让 AI 能够看到摄像头中的视频内容、听到用户说的话，并给予恰当回应。需综合考虑视觉内容理解准确性、语音交互自然度与流畅性，以及端云协同的成本控制策略。项目还需要提交设计文档，说明用户故事完成情况和成本控制策略。

## 比赛硬性规则

1. 代码仓库必须在本批次开题后创建。
2. 所有 commit 时间必须落在 6 月 12 日 00:00 到 6 月 14 日 23:59 之间。
3. 严禁最后一天一次性导入所有代码。
4. 必须保持持续 PR 和 commit 记录。
5. 每个 PR 只做一件事，只实现或修改一个明确功能。
6. PR 描述不能为空，必须包含：
   - 标题
   - 功能描述
   - 实现思路
   - 测试方式
7. 引用第三方库、框架、模型、API 时，必须在 README 中列明依赖，并说明原创功能部分。
8. 不要提交 .env、API Key、密钥、token。
9. 主分支 main 必须保持可运行状态。
10. 最终必须提交：
   - 公开可访问的 GitHub/Gitee 仓库
   - README 文档
   - Demo 视频链接
   - 设计文档

## 本项目开发原则

1. 优先完成可演示 MVP，不追求复杂架构。
2. 不要一次性生成完整项目。
3. 每次只做一个 PR 功能。
4. 每个功能完成后必须说明如何本地验证。
5. 不要新增与题目无关的功能，例如登录、复杂数据库、后台管理系统。
6. 所有功能必须围绕题目一：
   - 摄像头
   - 麦克风
   - 视觉理解
   - 语音对话
   - AI 回应
   - 成本控制
   - 设计文档

## 技术方向建议

优先实现以下 MVP 链路：

1. 前端打开摄像头。
2. 前端打开麦克风。
3. 用户说话后转成文本。
4. 用户提问时截取当前摄像头画面。
5. 后端接收图片和用户文本。
6. 后端调用多模态模型理解画面并生成回复。
7. 前端显示 AI 回复。
8. 前端朗读 AI 回复。
9. README 和设计文档说明端云协同和成本控制策略。

## 成本控制原则

实现时优先采用低成本策略：

1. 不连续上传完整视频流。
2. 用户提问时才截取当前画面。
3. 前端压缩图片后再上传。
4. 多轮对话复用最近一次视觉摘要。
5. 简单问题优先走文本模型。
6. 涉及画面问题时才调用视觉模型。
7. 设置请求节流，避免频繁调用模型。
8. API Key 只能从环境变量读取。
